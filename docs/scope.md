# Project Scope Statement

**Project Name:** Dahl Tech Solutions Website **Project Manager:** KayLee Dahl **Date:** February 15, 2026

## **1\. Project Purpose and Justification**

The purpose of this project is to design and develop a professional two-page website for a fictional small business, Dahl Tech Solutions. This business provides affordable IT support services for small schools and local businesses.

The website will serve as a digital presence that communicates services, builds credibility, and provides contact information for potential clients.

This project supports my academic growth in project management and web development while aligning with my professional goals of growing in IT and leadership roles. It will demonstrate my ability to manage a digital project from initiation through delivery, including documentation, planning, risk management, and technical execution.

## **2\. Project Objectives (SMART Goals)**

By the end of the semester:

* Create a fully functional two-page website (index.html and about.html).  
    
* Apply a single external CSS stylesheet (style.css) for consistent styling.  
    
* Publish the website using GitHub Pages.  
    
* Develop complete planning documentation including:  
    
  * A Project Scope Statement (this document)  
      
  * A Project Plan including risk analysis (TAME framework)  
      
  * A Project Retrospective


* Deliver a professional, organized, public GitHub repository that includes:  
    
  * Clear README.md  
      
  * Live site link  
      
  * Documentation folder

Success will be measured by:

* Meeting all minimum technical requirements  
    
* Maintaining scope discipline  
    
* Delivering by the project deadline  
    
* Creating a clean, professional user experience

## **3\. High-Level Requirements**

The final deliverable must include:

### **Repository Requirements**

* One public GitHub repository  
    
* Professional README.md file  
    
* Live website link (GitHub Pages)  
    
* docs/folder containing:  
    
  * scope.md  
      
  * plan.md  
      
  * retrospective.md

### **Website Requirements**

* index.html (homepage)  
    
* about.html (secondary page)  
    
* style.css (external stylesheet)  
    
* Functional navigation between pages  
    
* Responsive layout (basic responsiveness using CSS)  
    
* Clear content structure (header, navigation, main content, footer)

## **4\. High-Level Deliverables**

1. Public GitHub Repository  
     
2. Fully functional two-page website  
     
3. Project documentation (3 Markdown files)  
     
4. Live deployment via GitHub Pages

## **5\. Project Boundaries**

### **In Scope**

* Two static HTML pages  
    
* Basic CSS styling  
    
* Simple navigation  
    
* Informational content about fictional IT services  
    
* Deployment via GitHub Pages  
    
* Full planning documentation

### **Out of Scope**

* E-commerce functionality  
    
* Backend development  
    
* Contact form with server-side processing  
    
* Database integration  
    
* Advanced JavaScript functionality  
    
* Ongoing maintenance beyond submission

Scope discipline will be maintained to prevent feature creep.

## **6\. Assumptions**

* GitHub will remain accessible and functional.  
    
* GitHub Pages hosting will work as expected.  
    
* I have sufficient foundational HTML and CSS knowledge to complete the project.  
    
* The project timeline is manageable within my current academic and professional workload.

## **7\. Constraints**

* Must meet all minimum expectations outlined in the assignment.  
    
* Must be completed within the semester timeline.  
    
* Limited to HTML and CSS only.  
    
* Limited to two website pages.  
    
* Must be housed in a single GitHub repository.  
    
8. Risks (Preliminary Identification)

**Risk 1:** Time constraints due to work and family responsibilities **Impact:** Medium **Mitigation:** Schedule dedicated build sessions on weekends

**Risk 2:** Technical challenges with GitHub Pages deployment **Impact:** Medium **Mitigation:** Test deployment early

**Risk 3:** Scope creep (adding unnecessary features) **Impact:** Medium **Mitigation:** Follow approved scope document strictly

**Risk 4:** CSS formatting issues **Impact:** Low **Migitation:** Test frequently and review documentation
