## **Project Retrospective – Dahl Tech Solutions Website**

Project Manager: KayLee Dahl

### **1\. What Went Well**

The project benefited from strong upfront planning. Developing a clear scope statement early helped prevent feature creep and kept the website focused on assignment requirements. Breaking the project into phases made the work manageable alongside my professional and family responsibilities.

My technical execution went smoothly because I kept the website simple and professional rather than overly complex.

### **2\. What Went Wrong**

Time management was the biggest challenge. Balancing work, school, and family responsibilities required intentional scheduling. There were minor formatting challenges with CSS styling and responsiveness that required troubleshooting.

### **3\. What I Would Do Differently**

If I were managing this as a real client project, I would:

* Create wireframes before building

* Conduct basic user testing

* Include analytics tracking

* Add accessibility testing measures

I would also begin deployment testing earlier to reduce stress near the deadline.

### **4\. Key Takeaways**

This project strengthened my understanding of:

* Scope discipline

* Risk management

* Structured planning

* Technical execution within constraints

It reinforced the value of being a hybrid generalist: combining leadership, documentation, and technical skills to deliver a complete project package.
